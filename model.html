AI Model Architecture

Deep dive into our exoplanet detection model's design, performance, and training methodology.

Model Architecture

Neural network pipeline for exoplanet transit detection

Input Layer

6 features: Orbital Period, Transit Duration, Planetary Radius, Transit Depth, Stellar Magnitude, Equilibrium Temperature

Preprocessing

Normalization, feature scaling, and outlier detection

Hidden Layers

3 dense layers with ReLU activation (128, 64, 32 neurons)

Output Layer

3 classes: Confirmed Planet, Candidate, False Positive

Data Preprocessing

  • Min-Max normalization for numerical features
  • Outlier detection using IQR method
  • Feature correlation analysis
  • Missing value imputation

Network Architecture

  • Input layer: 6 neurons (features)
  • Hidden layer 1: 128 neurons + ReLU + Dropout(0.3)
  • Hidden layer 2: 64 neurons + ReLU + Dropout(0.2)
  • Hidden layer 3: 32 neurons + ReLU + Dropout(0.1)
  • Output layer: 3 neurons + Softmax

Training Configuration

  • Optimizer: Adam (lr=0.001)
  • Loss function: Categorical crossentropy
  • Batch size: 32
  • Early stopping with patience=10
  • Learning rate scheduling

Model Selection Rationale

The neural network approach was selected as our primary model due to its superior performance across all metrics. While it requires more computational resources and training time, the significant improvement in accuracy (94.7% vs 92.8% for the next best model) justifies the additional complexity. The model's ability to capture non-linear relationships in the exoplanet detection data makes it particularly well-suited for this application.
